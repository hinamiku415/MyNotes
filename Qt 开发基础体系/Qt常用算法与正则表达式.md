## Qt5 常用算法与正则表达式

在 Qt5 开发中，常用算法能快速解决数值计算类问题，正则表达式则是处理文本匹配、校验的强大工具。本文系统梳理 Qt5 核心常用算法、正则表达式的语法规则。

### 1. 核心常用算法详解

|    算法函数     |         功能描述          |               参数类型               |    返回值类型    |                  注意事项                  |
| :-------------: | :-----------------------: | :----------------------------------: | :--------------: | :----------------------------------------: |
|    `qAbs(a)`    |   计算数值 `a` 的绝对值   |   支持 `double`、`int` 等数值类型    | 与输入参数同类型 |        对负数取正，正数和零保持不变        |
|  `qMax(b, c)`   | 比较两个数值，返回较大值  | 两个同类型数值（如 `double`、`int`） | 与输入参数同类型 |    仅支持两个数值比较，多数值需嵌套使用    |
|   `qRound(b)`   | 对浮点数 `b` 进行四舍五入 |       `double` 或 `float` 类型       |    `int` 类型    |   小数部分 ≥0.5 向上取整，＜0.5 向下取整   |
| `qSwap(bn, cn)` |     交换两个变量的值      |      两个同类型变量（引用传递）      |     无返回值     | 直接修改原变量，支持数值、字符串等多种类型 |

### 2. 算法实战案例

#### 头文件（`widget.h`）

```cpp
#ifndef WIDGET_H
#define WIDGET_H

#include <QWidget>

class Widget : public QWidget
{
    Q_OBJECT

public:
    Widget(QWidget *parent = nullptr);
    ~Widget();
};

#endif // WIDGET_H
```

#### 源码文件（`widget.cpp`）

```cpp
#include "widget.h"
#include <QDebug>

Widget::Widget(QWidget *parent)
    : QWidget(parent)
{
    // 初始化测试数值
    double x = -98.654;  // 负数浮点数
    double y = 25.98;    // 正数浮点数

    // 1. 计算绝对值
    double result1 = qAbs(x);
    qDebug() << "qAbs(-98.654) = " << result1;  // 输出：98.654

    // 2. 获取两个数的最大值
    double maxResult = qMax(x, y);
    qDebug() << "qMax(-98.654, 25.98) = " << maxResult;  // 输出：25.98

    // 3. 浮点数四舍五入为整数
    int result2 = qRound(y);
    int result3 = qRound(x);
    qDebug() << "qRound(25.98) = " << result2;  // 输出：26（0.98≥0.5 向上取整）
    qDebug() << "qRound(-98.654) = " << result3;  // 输出：-99（0.654≥0.5 向上取整为-99）

    // 4. 交换两个变量的值
    qDebug() << "\n交换前 x = " << x << ", y = " << y;  // 输出：-98.654, 25.98
    qSwap(x, y);
    qDebug() << "交换后 x = " << x << ", y = " << y;  // 输出：25.98, -98.654
}

Widget::~Widget()
{
}
```

#### 运行结果

```json
qAbs(-98.654) =  98.654
qMax(-98.654, 25.98) =  25.98
qRound(25.98) =  26
qRound(-98.654) =  -99

交换前 x =  -98.654 , y =  25.98
交换后 x =  25.98 , y =  -98.654
```

## 二、正则表达式：文本匹配与校验的利器

正则表达式（Regular Expression，简写为 regex、regexp 或 RE）是一种文本模式，通过普通字符和特殊字符（元字符）组合，描述一系列符合语法规则的字符串。常用于文本检索、替换、格式校验等场景，Qt5 支持标准 C++ 正则表达式库（`<regex>`），也可结合 Qt 自身字符串类使用。



### 1. 正则表达式核心组成

正则表达式由 **表达式（expressions）**、**量词（quantifiers）**、**断言（assertions）** 三部分组成，以下是 Qt 开发中最常用的语法规则：

#### （1）基础表达式（字符匹配）

|         表达式          |                           描述                            |                   示例                    |
| :---------------------: | :-------------------------------------------------------: | :---------------------------------------: |
| 普通字符（a-z、0-9 等） |                         匹配自身                          |          `abc` 匹配字符串 "abc"           |
|         `[xyz]`         |             字符集合，匹配任意一个包含的字符              |   `[AEIOU]` 匹配大写元音字母 A/E/I/O/U    |
|        `[^xyz]`         |            负值字符集合，匹配不包含的任意字符             | `[^AEIOU]` 匹配非大写元音字母（辅音字母） |
|         `[a-z]`         |            字符范围，匹配指定范围内的任意字符             |  `[0-9]` 匹配 0-9 任意数字，等价于 `\d`   |
|        `[^a-z]`         |          负值字符范围，匹配不在指定范围内的字符           |   `[^0-9]` 匹配非数字字符，等价于 `\D`    |
|          `\d`           |               匹配数字字符，等价于 `[0-9]`                |         `\d{3}` 匹配 3 个连续数字         |
|          `\D`           |              匹配非数字字符，等价于 `[^0-9]`              |      `\D+` 匹配一个或多个非数字字符       |
|          `\w`           | 匹配单词字符（字母、数字、下划线），等价于 `[A-Za-z0-9_]` |       `\w+` 匹配一个或多个单词字符        |
|          `\W`           |          匹配非单词字符，等价于 `[^A-Za-z0-9_]`           |      `\W` 匹配空格、标点等非单词字符      |
|          `\s`           |          匹配空白字符（空格、制表符、换页符等）           |       `\s+` 匹配一个或多个空白字符        |
|          `\S`           |                      匹配非空白字符                       |      `\S*` 匹配零个或多个非空白字符       |
|           `.`           |     匹配除换行符外的任意单个字符（Qt 中可匹配换行符）     |        `a.b` 匹配 "aab"、"acb" 等         |
|           `|`           |               逻辑或，匹配左侧或右侧表达式                |           `a|b` 匹配 "a" 或 "b"           |

#### （2）量词（匹配次数）

|   量词   |               含义                | 等价写法 |                      示例                       |
| :------: | :-------------------------------: | :------: | :---------------------------------------------: |
|   `E?`   |      匹配表达式 E 零次或一次      | `E{0,1}` |             `ab?` 匹配 "a" 或 "ab"              |
|   `E+`   |      匹配表达式 E 一次或多次      | `E{1,}`  |            `ab+` 匹配 "ab"、"abb" 等            |
|   `E*`   |      匹配表达式 E 零次或多次      | `E{0,}`  |         `ab*` 匹配 "a"、"ab"、"abb" 等          |
|  `E{n}`  |      匹配表达式 E 恰好 n 次       |    -     |            `\d{3}` 匹配 3 个连续数字            |
| `E{n,}`  |      匹配表达式 E 至少 n 次       |    -     |        `\d{6,}` 匹配 6 个及以上连续数字         |
| `E{n,m}` | 匹配表达式 E 至少 n 次、最多 m 次 |    -     | `\d{6,18}` 匹配 6-18 个连续数字（密码长度常用） |

#### （3）断言（位置匹配）

|  符号   |                 含义                  |                             示例                             |
| :-----: | :-----------------------------------: | :----------------------------------------------------------: |
|   `^`   |          匹配字符串开头位置           |                 `^1` 匹配以 "1" 开头的字符串                 |
|   `$`   |          匹配字符串结尾位置           |                 `\d$` 匹配以数字结尾的字符串                 |
|  `\b`   | 单词边界（单词与空格 / 标点的分隔处） |    `er\b` 匹配 "never" 中的 "er"，不匹配 "verb" 中的 "er"    |
|  `\B`   |              非单词边界               |    `er\B` 匹配 "verb" 中的 "er"，不匹配 "never" 中的 "er"    |
| `(?=E)` |  正向预查：表达式后必须紧随 E 才匹配  |      `const(?=\s+char)` 匹配 "const char" 中的 "const"       |
| `(?!E)` |  负向预查：表达式后不能紧随 E 才匹配  | `const(?!\s+int)` 匹配 "const char" 中的 "const"，不匹配 "const int" 中的 "const" |

#### （4）转义字符

| 转义字符  |                           描述                            |          示例          |
| :-------: | :-------------------------------------------------------: | :--------------------: |
|   `\\`    | 匹配反斜杠本身（需双重转义，Qt 字符串中用 `\\` 表示 `\`） | 匹配 "\" 需写为 `\\\\` |
| `\(` `\)` |                      匹配圆括号本身                       | `\(abc\)` 匹配 "(abc)" |
|   `\n`    |                        匹配换行符                         |           -            |
|   `\t`    |                        匹配制表符                         |           -            |

### 2. 正则表达式实战案例：手机号码校验

#### 需求说明

根据国内手机号码规则，校验字符串是否为合法手机号：

- 总长度 11 位数字；
- 以 1 开头；
- 第 2 位为 3、5、8（覆盖移动、联通、电信常见号段）；
- 后 9 位为任意数字。

#### 头文件（mainwindow.h）

```cpp
#ifndef MAINWINDOW_H
#define MAINWINDOW_H

#include <QMainWindow>

class MainWindow : public QMainWindow
{
    Q_OBJECT

public:
    MainWindow(QWidget *parent = nullptr);
    ~MainWindow();
};

#endif // MAINWINDOW_H
```

#### 源码文件（mainwindow.cpp）

```cpp
#include "mainwindow.h"
#include <QDebug>
#include <regex>
#include <QString>

MainWindow::MainWindow(QWidget *parent)
    : QMainWindow(parent)
{
    // 测试手机号（合法/非法）
    QString validPhone = "18923558899";    // 合法手机号
    QString invalidPhone1 = "12345678901"; // 第2位为2，非法
    QString invalidPhone2 = "1381234567";  // 长度10位，非法
    QString invalidPhone3 = "138123456789";// 长度12位，非法
    QString invalidPhone4 = "a13812345678";// 包含非数字字符，非法

    // 正则表达式：^1(3|5|8)\\d{9}$
    // 解析：
    // ^1：匹配以1开头
    // (3|5|8)：第2位为3、5、8中的任意一个
    // \\d{9}：匹配9个连续数字（Qt字符串中需双重转义，\\d 表示 \d）
    // $：匹配字符串结尾，确保总长度11位
    std::regex phoneReg("^1(3|5|8)\\d{9}$");

    // 定义校验函数（简化代码）
    auto checkPhone = [&](const QString &phone)  //[&]:捕获当前作用域中所有 Lambda 表达式外部的变量
    {
        std::string phoneStr = phone.toStdString();
        bool isMatch = std::regex_match(phoneStr, phoneReg);
        if (isMatch) {
            qDebug() << phone << " --> 合法手机号码";
        } else {
            qDebug() << phone << " --> 非法手机号码";
        }
    };

    // 校验测试
    checkPhone(validPhone);
    checkPhone(invalidPhone1);
    checkPhone(invalidPhone2);
    checkPhone(invalidPhone3);
    checkPhone(invalidPhone4);
}

MainWindow::~MainWindow()
{
}
```

#### 运行结果

```cpp
"18923558899"  --> 合法手机号码
"12345678901"  --> 非法手机号码
"1381234567"  --> 非法手机号码
"138123456789"  --> 非法手机号码
"a13812345678"  --> 非法手机号码
```

## 三、关键注意事项

1. **Qt 字符串转义问题**：Qt 中字符串的 `\` 是转义字符，因此正则表达式中的 `\d`、`\b` 等需写为 `\\d`、`\\b`（双重转义），避免被 Qt 解析为其他含义。
2. **正则匹配范围**：`std::regex_match` 匹配整个字符串（需满足完整正则表达式），`std::regex_search` 匹配字符串中任意符合的子串，根据需求选择合适的匹配函数。
3. **算法参数类型**：`qMax`、`qSwap` 等算法要求参数类型一致，若需比较不同类型（如 `int` 和 `double`），需先进行类型转换。
4. **正则表达式可读性**：复杂正则表达式建议添加注释，或拆分为多个部分组合，提升可维护性（如手机号正则可拆分为前缀、中间段、后缀）。
