## Qt 信号与槽机制实战应用

在 Qt 开发中，信号与槽（Signal & Slot）机制是核心通信方式，基于观察者模式设计，能实现对象间松散耦合的交互。

## 一、信号与槽机制核心原理

### 1. 核心概念定义

- **信号（Signal）**：本质是事件的抽象表达，以函数形式声明。当特定事件发生时（如按钮点击、数据变化），对象会发出对应的信号。**信号函数仅需声明，无需实现**。
- **槽（Slot）**：是响应信号的函数，与普通函数的核心区别是可与信号关联，当关联信号被发射时，槽函数会自动执行。**槽函数需完整定义（实现）**，且可通过`public slots/protected slots/private slots`修饰以控制访问权限。
- **关联核心**：通过`QObject::connect()`静态函数建立信号与槽的关联，是信号与槽机制的核心执行入口。

**注意**：`signals`和`slots`是 Qt 在 C++ 基础上扩展的专属关键词，专门用于标识信号函数和槽函数，是 Qt 信号与槽机制的语法基础

### 2. connect () 函数详解

```cpp
[static] QMetaObject::Connection QObject::connect(const QObject *sender, const char *signal, const QObject *receiver, const char *method, Qt::ConnectionType type = Qt::AutoConnection)
```

**参数说明**

- `sender`：发出信号的对象（信号发送者）；
- `signal`：发送者对象要发出的具体信号；
- `receiver`：接收信号的对象（信号接收者）；
- `method`：接收者对象中用于响应信号的槽函数，信号触发后会自动调用该方法；
- `type`：连接类型，默认值为`Qt::AutoConnection`，用于控制信号传递的线程行为等

## 二、信号与槽的 3 种核心连接方式

Qt 信号与槽支持灵活的连接组合，满足不同场景下的通信需求：

### **1. 信号 - 信号连接**

一个信号可直接与另一个信号相连，**实现信号的转发与连锁触发**。

```cpp
connect(object1, SIGNAL(signal1), object2, SIGNAL(signal1)); 
```

- 核心逻辑：当`object1`发出`signal1`信号时，会自动触发`object2`的`signal1`信号，进而联动所有与`object2->signal1`关联的槽函数。

### 2. 信号 - 多槽连接

同一个信号可关联多个槽函数，**实现 “一信号多响应**”。

```cpp
connect(object1, SIGNAL(signal2), object2, SLOT(slot2));
connect(object1, SIGNAL(signal2), object3, SLOT(slot1));
```

- 核心逻辑：`object1`发出`signal2`信号时，`object2`的`slot2`和`object3`的`slot1`会同时被触发执行

### 3. 多信号 - 槽连接

同一个槽函数可响应多个不同信号，**实现 “多事件同一处理”**

```cpp
connect(object1, SIGNAL(signal2), object2, SLOT(slot2));
connect(object3, SIGNAL(signal2), object2, SLOT(slot2));
```

- 核心逻辑：无论`object1`还是`object3`发出`signal2`信号，都会触发`object2`的`slot2`槽函数执行。

### 4.常用连接方案

```cpp
connect(object1, SIGNAL(signal), object2, SLOT(slot));
```

## 三、信号与槽机制的优势与注意事项

### 1. 核心优势：松散耦合

信号发送者无需知道接收者的具体实现，也无需了解接收者是否存在对应的槽函数；接收者仅需关注关联的信号，无需依赖发送者的具体逻辑。这种设计大幅降低了对象间的耦合度，提升了代码的可维护性和扩展性。

### 2. 使用前提

支持信号与槽机制的类，**必须直接或间接继承自`QObject`类**；且 **Qt 信号与槽机制不支持模板的使用**。

### 3. 效率

信号与槽机制虽灵活，但相比直接调用非虚函数，运行速度会稍慢，主要原因包括：

- 多线程场景下，信号可能需要排队等待；
- 信号传递参数时需进行编组与解组操作；
- 需安全遍历所有关联的信号 - 槽对；
- **需定位信号接收对象并调用对应的槽函数**

## 代码实例，写一个任意圆半径体积计算器

```cpp
#include "dialog.h" //前提都在h里面定义与声明完毕

const static double PI = 3.1415;
#include <QGridLayout>  //表格布局
Dialog::Dialog(QWidget *parent)
    : QDialog(parent)
{
    //创建一个标签（提示输入圆球半径）
    lab1 = new QLabel(this);
    lab1->setText(tr("输入圆球半径："));

    // 创建第二个标签，显示圆球体积结果
    lab2 = new QLabel(this);

    //编辑框控件（接收圆球半径）
    lEdit = new QLineEdit(this);

    //创建命令按钮
    pbt = new QPushButton(this);
    pbt->setText(tr("计算圆球体积"));

    //表格布局
    QGridLayout *mLay = new QGridLayout(this);
    mLay->addWidget(lab1,0,0);
    mLay->addWidget(lEdit,0,1);
    mLay->addWidget(lab2,1,0);
    mLay->addWidget(pbt,1,1);

    
    connect(lEdit,SIGNAL(textChanged(QString)),this,SLOT(CalcBallVolume()));
    //这里绑定了lEdit，意思就是主要输入框里的数字发生改变，那么就会发送信号给当前这个类，执行计算体积
}

Dialog::~Dialog() {}


void Dialog::CalcBallVolume() //槽函数
{
    bool isLoop;
    QString tempSTr;
    QString valueStr = lEdit->text();

    int valueInt = valueStr.toInt(&isLoop);
    double dVSum = 4.0/3.0*PI*valueInt*valueInt;

    lab2->setText(tempSTr.setNum(dVSum));

}
```

**核心**：**将名为 `lEdit` 的输入框控件的 `textChanged(QString)` 信号，连接到当前类（`this`）的 `CalcBallVolume()` 槽函数**。当用户修改 `lEdit` 中的文本时，会自动触发 `CalcBallVolume()` 函数执行实时计算球体体积。

同时推荐另一种写法：使用**新的基于函数指针的写法**（类型更安全，编译期就能发现错误）

```cpp
// Qt5/Qt6 推荐写法（类型安全）
connect(lEdit, &QLineEdit::textChanged, this, &当前类名::CalcBallVolume);
```
